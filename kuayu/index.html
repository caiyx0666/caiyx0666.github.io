前端跨域问题详解

作为一名前端工程师，在面试的时候你是否经常被问到：什么是跨域？怎么解决跨域？

这是两个高频面试题，今天这篇文章将为你带来详细的介绍什么是跨域，如何解决跨域的问题

什么是跨域

跨域其实也叫做非同源策略请求。谈到跨域问题，我们必须先了解一个什么是同源策略

同源策略：它是由Netscape提出的一个著名的安全策略。现在所有支持JavaScript的浏览器都会使用这个策略。所谓同源指的是，域名、协议、端口相同。其中有一个不相同就是跨域

跨域的几种解决方案

JSONP实现跨域

jsonp的原理就是利用 <script> 标签没有跨域限制，通过 script 标签 src 属性，发送带有callback回调函数的GET请求，服务端将接口返回数据拼凑到callback函数中，返回给浏览器，浏览器解析执行，从而让前端拿到callback函数返回的数据

jsonp和ajax对比

jsonp和ajax相同，都是客户端向服务器端发送请求，从服务端获取数据的方式。但是ajax属于同源策略，jsonp属于非同源策略（跨域请求）

jsonp优缺点

优点：简单兼容性好，可用于解决主流浏览器的跨域数据访问的问题

缺点：仅支持发送GET请求，且不安全，容易遭受XSS攻击

jsonp的实现流程

  1. 声明一个回调函数（如getDate），函数名当作参数，传递给跨域请求数据的服务器，函数形参为要获取的目标数据（服务器返回的data）
  2. 创建一个 <script> 标签，把那个跨域的API数据接口地址，赋值给<script>的src属性,还要在这个地址中向服务器传递该函数名（可以通过问号传参?callback = getData）
  3. 服务器接收到请求后，需要进行特殊的处理：把传递过来的函数名和它需要给你返回的数据拼接成一个字符串，例如：传递进去的函数名是getData，它准备好的数据是getData(‘返回的数据’)
  4. 最后服务器把准备的数据通过HTTP协议返回给客户端，客户端解析执行，就能够对应返回的数据进行操作。

我们的ajax一样是支持JSONP的，但是同样需要服务端配合，服务端代码同上即可，只要是调用callback函数，并返回相应的数据就即可

CORS实现跨域

CROS（Cross-Origin Resource Sharing）跨源资源分享。

CORS实现跨域是现在比较常用的跨域解决方案。CROS允许浏览器向跨源服务器发出XMLHttpRequest请求，克服了ajax不能跨源请求的限制。

CORS实现跨域主要在于服务器，只要服务器实现CROS接口，就可以实现跨域请求。客户端只需要正常发送就可以了，效果跟发送同源请求是一样的。

浏览器在头信息中增加一个Origin字段，从而发出CROS请求 要支持CORS访问需要服务器在响应头中添加Access-Control-Allow-Origin，可以使用*来表示允许所有域跨域访问。
